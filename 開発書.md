# 自動車鈑金塗装工場向け 工程管理・カンバンボードアプリ 開発要件書

---

## 1. プロジェクト概要

自動車の鈑金・塗装工場における車両の工程管理、および代車・レンタカーの貸出状況を管理するWebアプリケーションです。

JiraのUI/UXを参考に、直感的なドラッグ＆ドロップ操作でタスク（カード）を移動できるカンバンボードと、車両の貸出状況を視覚的に把握できるガントチャートを提供します。

### 主な技術スタック

| 項目 | 技術 |
|------|------|
| Frontend Framework | React (Function Components / Hooks) |
| Styling | Tailwind CSS |
| Icons | lucide-react |
| ビルドツール | Vite または Create React App（Cursorの環境に合わせる） |

---

## 2. コア機能要件

### 2.1 ログイン画面

- **Googleアカウントでのログイン**ボタン（UIのみのモックアップ可。将来的にFirebase Auth等と連携予定）
- ログイン完了後、メインのカンバンボード画面へ遷移する

### 2.2 全体レイアウト・ナビゲーション

**ヘッダー**

- プロジェクト名、ナビゲーションリンク（ダッシュボード、カード、ボード等）
- 「ボード」切り替え用ドロップダウンメニュー
- 「カード作成」ボタン
- 検索バー、通知、ユーザーアイコン

**左サイドバー（折りたたみ）**

- 「カンバンボード」ビューへの切り替えアイコン
- 「代車ガントチャート」ビューへの切り替えアイコン

### 2.3 カンバンボード機能 (Board View)

**複数ボードの切り替え**

- 以下の5種類のボードを定義し、ヘッダーまたはタイトル横のメニューから切り替え可能にする
  - Planning（入庫予約管理）
  - Main（全作業工程）
  - Paint（塗装工程）
  - Delivery（納車管理）
  - Body（鈑金工程）
- 選択したボードに応じて、表示される「列（ステータス）」を動的に変更する

**カード（タスク）の表示**

- 車種、車番、顧客名、入庫日・納車日を表示
- カードの色（ヘリにカラーバー、または背景色）
- 4つのステータスドット色
- キャラクターアイコン（車、ローラー、レンチなど）、タスクアイコン（チェック、ファイルなど）
- 代車が必要な場合は「トラックアイコン＋代」のバッジを表示

**ドラッグ＆ドロップ**

- HTML5のDnD APIを使用し、カードを別の列（ステータス）へドラッグ＆ドロップで移動可能にする

### 2.4 代車・レンタカー ガントチャート機能 (Gantt View)

**レイアウト**

- 左側に車両一覧（フリート）、右側に日付軸のタイムライン

**日付軸**

- 今日を起点に、3日前から数週間分の日付（月/日 曜日）を表示
- 土日は背景色を変える

**車両データ**

- 軽自動車、普通車、レンタカー枠などの車両マスターを定義

**予約バーの表示と操作**

- 予約データ（開始日、終了日、顧客・車種、使用車両）に基づき、タイムライン上に横バーを描画する
- **ドラッグ＆ドロップでの予定変更**: 予約バーをドラッグし、別の日付や別の車両の行へドロップすることで、予約の「開始日・終了日（期間は維持）」および「割り当て車両」を直感的に変更できること

### 2.5 カード（タスク）作成モーダル

モーダルウィンドウ内で以下の項目を入力・選択できること。

- **アイコン選択**: 複数のキャラクター/タスクアイコンから複数選択可能
- **日付**: 入庫日（デフォルトは本日）、納車日
- **代車・レンタカー**: 不要、代車(軽)、代車(普通)、レンタカー から選択
- **車両情報**
  - メーカー（プルダウン選択）
  - モデル（メーカーに連動したプルダウン選択）
  - 車番（テキスト）、顧客名（テキスト）
- **ビジュアル設定**: 4つのドット色、カードの背景色
- **詳細**: 担当者、説明（テキストエリア）、PDF添付ファイル（ドラッグ＆ドロップ対応のUI）

### 2.6 カード詳細パネル（スライドイン）

- カンバンボード上でカードをクリックすると、画面右側から詳細パネルがスライドインする
- カンバンボードの幅はパネルの分だけ自動的に縮むこと

**インライン編集**

- 代車利用、車種、車番、顧客名、入庫日、納車日、説明などの項目を、パネル内で直接（input や textarea を用いて）編集できる
- 変更内容は即座に state に反映され、カンバンボード上のカード表記も連動して更新されること
- ドット色やカード色、表示アイコンの変更もパネル内から可能

## 4. 実装上の注意点とCursor AIへの指示のコツ

### シングルファイル構成（プロトタイプ段階）

初期段階では、コンポーネントの挙動を素早く確認するため、**App.jsx などの1つのファイルに全コンポーネント**（LoginScreen, KanbanApp, LoanerGanttChart, CreateTaskModal, TaskDetailPanel）をまとめて実装してください。

### スタイリング

Tailwind CSSのユーティリティクラスを最大限活用し、CSSファイルへの記述は最小限に留めてください。

### ドラッグ＆ドロップ実装

外部ライブラリ（react-beautiful-dnd等）は使わず、**ネイティブのHTML5 Drag and Drop API**（onDragStart, onDragOver, onDrop）を用いて軽量に実装してください。

### イベント伝播

モーダルやパネルの外側をクリックした際の「閉じる」挙動には、`useOutsideClick` のようなカスタムフックを活用してください。

### Cursor Composerへの初期プロンプト例

```
@README.md の要件定義に従って、ReactとTailwind CSSを使用した工程管理アプリを作成してください。
まずは App.jsx 1つのファイルに、モックデータと全コンポーネントを含めた完全なコードを生成してください。
lucide-reactのアイコンを使用し、要件にあるDnDでのステータス移動やガントチャートのバー移動機能も実装してください。
```
