# 清田自動車 工程管理アプリ NFC列変更仕様

---

## 1. 概要

- **目的**
  - 各車両の作業指示書バインダーに貼り付けた **NFCタグから、対応するカードの現在列を確認し、移動先の列をワンタップで変更できる** 仕組みを定義する。
  - 現場の作業者が、PC を操作せずに **スマートフォン＋NFC** だけでステータス変更できることを目標とする。

- **対象**
  - フロントエンド（React + Vite）
  - 認証基盤: Firebase Authentication（Google）
  - データ保存: Cloud Firestore（`boards/main/tasks` コレクション）

---

## 2. NFCタグとURL

### 2.1 NFCタグに書き込む内容

- NFCタグには **URL 文字列のみ**を書き込む。
- URL 形式（例）:
  - `https://{本番ドメイン}/kiyota/?nfcTaskId={カードID}`
- `nfcTaskId` には、工程管理カードの `id`（例: `t1689754321123`）を埋め込む。
- バインダーごとに **1つのカードIDと1つのNFCタグ**を紐付ける運用とする。

### 2.2 URL 発行方法（運用想定）

- カードID は Firestore 上で一意に管理されている。
- 現状は、ブラウザで対象カードを選択した状態でアドレスバーの URL を確認し、  
  `?nfcTaskId=...` を付与した URL を NFC 書き込みアプリに手動で入力する運用とする。
- 将来的に、カード詳細画面に「NFC用URLをコピー」ボタンを追加する拡張余地を残す。

---

## 3. アプリ側の動作仕様

### 3.1 URLパラメータの取り扱い

- アプリ起動時（`App` コンポーネントの `useEffect`）で、次を行う。
  - クエリ文字列から `nfcTaskId` を取得し、存在すれば **`nfcTaskId` ステートに保存**する。
  - 取得後は URL から `nfcTaskId` と `fromCalendar` を削除し、  
    `history.replaceState` でクリーンな URL に置き換える（更新しても同じURLで開けるようにする）。
- ログイン前に NFC URL を開いた場合でも、ログイン完了後に `nfcTaskId` が反映される。

### 3.2 KanbanApp での NFC モード

- `App` から `KanbanApp` に `nfcTaskId` を `props` で渡す。
- `KanbanApp` 側では次の挙動をとる。
  - 初期 `selectedTaskId` を `nfcTaskId` で初期化する。
  - `useEffect` で `nfcTaskId` が設定されたとき、
    - `currentBoardId` を `main`（〈全作業〉工程管理）に設定。
    - `currentView` を `board` に設定。
    - `selectedTaskId` に `nfcTaskId` を再セット（対象カードを自動選択）。

### 3.3 NFCモード用 UI（列変更パネル）

- 全作業ボードのカンバン上部（列一覧の直上）に、`nfcTaskId` が有効な場合のみ **NFCモード用バナー**を表示する。
- バナー内容:
  - 左側:
    - ラベル: 「NFCモード: 列を選んで移動」
  - 右側:
    - 対象カードの概要: `{車種} {車番} / {顧客名}`（縮小表示）
  - 下部:
    - 現在のボード（`main`）に属する全列（`boardColumns`）を、**小さなボタン群**として横並びに表示する。
    - 各ボタンのラベルは `col.name`（例: 「入庫済み」「B待ち」「B中」…）。
    - 現在列に対応するボタンのみ、色を変えて強調表示する。

### 3.4 列変更の動作

- NFCモードの列ボタンが押されたとき:
  - 対象列の「代表ステータス」（`getColumnPrimaryStatus(col)` で取得）を決定する。
  - そのステータスが、現在のカード `status` と異なる場合のみ更新を行う。
  - `transitionTaskStatus` を用いて、ステータス履歴（`statusHistory`）と `statusEnteredAt` を更新しつつ、新しい `task` オブジェクトを生成する。
  - `setTasks` でローカル状態を更新し、同時に `upsertDocument('boards/main/tasks', updatedTask.id, updatedTask)` で **Firestore にも書き込む**。
  - Firestore の `onSnapshot` により、他のPC・ブラウザでもほぼリアルタイムに列変更が反映される。

### 3.5 その他の仕様

- NFCモードであっても、通常どおりカードのドラッグ＆ドロップや詳細編集は可能。
- NFCモードのバナーは、`selectedTaskId` が存在し、かつ `nfcTaskId` がセットされている場合のみ表示される。
- `nfcTaskId` で指定されたカードが存在しない場合:
  - `selectedTask` が `null` となり、NFCバナーは表示されない。
  - アプリは通常のカンバン画面として動作する（エラー画面にはしない）。

## 4. 想定利用シナリオ

1. 事務スタッフが PC で工程管理を開き、カードを作成／編集する。
2. 対応するカードID を使って NFCタグ用 URL を作成し、NFCタグに書き込む。
3. バインダーに NFCタグを貼り付ける。
4. 作業担当者はスマートフォンを NFCタグにかざし、ブラウザで URL を開く。
5. ログインしていなければ、Googleログイン画面が表示されるので認証する。
6. ログイン後、全作業ボードが開き、対象カードが選択されるとともに、画面上部に NFCモードバナーが表示される。
7. 作業担当者は「B待ち」「B中」「本日納車」など、移動先列のボタンをタップするだけでステータスを変更できる。
8. 変更は Firestore を介して共有され、事務PC・他の端末のカンバンにも即時反映される。

---

## 5. 制約・注意点

- 現時点では、NFC経由で変更できる列は **「〈全作業〉工程管理（Main）」の列** のみとする。
- NFCタグの URL は **本番ドメイン前提** で運用する（ローカルホスト用 URL は別扱い）。
- 認証・許可は Google ログイン＋ Firebase Authentication の仕様に従う。
- iOS のブラウザ（特に Chrome）では、認証周りの制約により挙動が不安定になる場合があるため、実運用では **Safari 推奨** とする。
